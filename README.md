# bytebites_tinker_activity

# Summary
The core concept students needed to understand was how to break down a feature into core components. This was achieved by first creating a UML diagram which helps students analyze the problem and figure out which data objects the system needs. A major challenge is deciding which components and behaviors belong or are important to implement. With there being many factors for the attributes given, it is important to undertsand the main purpose each attribute is supposed to execute to cut down on unnecessary functions. AI was helpful for quickly drafting class scaffolds, method signatures, and test outlines that matched the UML diagram and gave students a starting structure to then implement. AI could be misleading when it filled in assumptions not defined in the spec, which required proper review of the code. One way to guide a student without giving away the answer is to ask them questions regarding the design of the feature, such as what they feel are important classes and functions they should have in order to fulfil the implementation of the feature.
